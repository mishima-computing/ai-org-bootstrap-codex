Apply-only: applies solely when an objective explicitly declares ui-retro-gamer; never inferred from nostalgia, pixels, or RPG labels.
Admissibility: a retro/game-feel term is admissible only when mapped to an observable event trigger, the state/proof it establishes, a guard/failure path, a render or scene-command hook, a timing cadence band, a sound-or-silence fallback, and a verification or explicit gap.
Contract-form: each kept term names (event, state-it-proves, guard, render-hook, cadence-band, silent-fallback, verification). A bare adjective ("punchy", "juicy", "nostalgic") with no observable mapping is rejected, not implemented.
Input: input/message cadence must be bounded, replayable, or explicitly marked as a gap; cite anchor:motion#nng-animation-duration for duration bands.
Reveal: a gacha/reveal ceremony separates anticipation, rarity signal, and item identity, each as an observable state transition with a skip and reduced-motion path; cite anchor:motion#wcag22-motion-criteria and anchor:genre-gacha#yin-xiao-chi2022-rrm.
Fallback: every animated/audio beat carries a silent / reduced-motion / no-audio fallback that is itself an observable state, not an absence.
Evidence: prefer runner/replay evidence — an input trace, a message-cadence proof, a state-transition log, a GUI/test spec, or a manual proof object; absence is declared as a gap, never assumed.
Risk: retro/game-feel grammar is taste-laden and stale-prone; treat as short-horizon until a product Stage-A spec ratifies it.
Pointers: federated motion anchors for cadence/reduced-motion and the RRM reveal-experience research; genre grammar stays card-owned, anchor-supported.
